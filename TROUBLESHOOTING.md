# 故障排除指南

## 问题：图片没有加载（显示洋红色方块和蓝色圆圈）

### 症状
- 农田显示为洋红色/紫红色方块
- 角色仍然是蓝色圆圈
- 没有看到真实的图片资源

### 解决方案

#### 方法1：清除缓存并重新运行（推荐）
1. 关闭正在运行的游戏
2. 双击运行 `重新启动游戏.bat`
3. 或者手动执行以下步骤：
   - 删除 `__pycache__` 文件夹
   - 运行 `python main.py`

#### 方法2：直接从命令行运行
```bash
# 切换到游戏目录
cd D:\桌面\爆炸\farm_game

# 清除缓存
rmdir /s /q __pycache__

# 运行游戏
python main.py
```

#### 方法3：检查控制台输出
运行游戏时，查看控制台输出中是否有：
```
==================================================
开始加载游戏资源...
==================================================
正在加载角色动画...
  ? 加载 down: 4 帧
  ? 加载 down_idle: 2 帧
  ...
```

如果看到 "加载图片失败" 的错误信息，说明图片文件有问题。

### 常见原因

1. **Python缓存问题** ?最常见
   - Python使用旧的 `.pyc` 文件
   - 解决：删除 `__pycache__` 文件夹

2. **graphics文件夹位置错误**
   - 确认 `graphics` 文件夹在 `farm_game` 目录下
   - 结构应该是：
     ```
     farm_game/
     ├── main.py
     ├── graphics/
     │   ├── character/
     │   ├── soil/
     │   └── ...
     └── ...
     ```

3. **Pygame没有正确初始化**
   - 确保安装了pygame: `pip install pygame`

4. **图片文件损坏**
   - 检查 `graphics` 文件夹中的PNG文件是否完整

### 验证资源加载

运行测试脚本：
```bash
python test_resources.py
```

这会显示详细的资源加载信息。

### 如果问题仍然存在

1. 查看游戏启动时的控制台输出
2. 截图发送控制台输出
3. 确认 `graphics` 文件夹完整性

### 成功的标志

游戏正确加载后，你应该看到：
- ? 角色是动画精灵（不是蓝色圆圈）
- ? 农田有土壤纹理（不是洋红色）
- ? 作物有生长阶段图片
- ? 周围有树木、灌木等装饰
